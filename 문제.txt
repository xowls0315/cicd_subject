📦 Nest.js 기반 CI/CD + Docker 배포 자동화 과제
 
🎯 과제 목적
이 과제의 목적은 Nest.js 애플리케이션을 기준으로 실제 현업에서 사용되는 배포 자동화 흐름을 직접 구현해보는 것입니다.
단순히 서버를 실행하는 수준을 넘어, 다음과 같은 DevOps 핵심 역량을 검증합니다.

Nest.js 기반 REST API 설계 능력
Lint / Test / Build를 포함한 CI 파이프라인 구성 능력
Docker 기반 컨테이너화(Containerization) 이해
docker-compose를 이용한 멀티 컨테이너 환경 구성
코드 → 빌드 → 배포로 이어지는 자동화 흐름에 대한 구조적 이해
⚠️ 이 과제는 “기능 구현 과제”가 아니라
**“개발 → 검증 → 배포 자동화 전체 흐름을 설계하는 과제”**입니다.

 
🧱 기본 조건 (중요)
제공되는 리소스는 없습니다
최초 상태는 Nest.js 기본 프로젝트(app 모듈만 존재) 입니다
추가 라이브러리 설치는 필요한 경우에 한해 허용됩니다
불필요한 기능, 과도한 설정은 감점 요인입니다
 
1️⃣ API 구현 요구사항 (Nest.js)
📍 기본 조건
HTTP Method는 GET만 사용
Controller 기반으로 라우터를 구성할 것
반환 값은 단순 문자열(String)
📍 필수 라우터 목록
URL 경로	응답 문자열
/hello	Hello World
/hello/docker	Hello Docker
/hello/cicd	Hello CI/CD
📌 주의사항

응답 포맷(JSON 등)은 사용하지 않습니다
Service 분리 여부는 자유이나, Nest.js 구조를 훼손해서는 안 됩니다
 
2️⃣ CI/CD 파이프라인 구성 (YAML)
📍 목적
코드 변경 시 자동으로 품질 검증 → 빌드 검증이 이루어지는 CI 파이프라인을 구성합니다.

📍 필수 포함 단계
CI 파이프라인에는 반드시 다음 단계가 포함되어야 합니다.

Lint
ESLint 기반
문법 오류 및 코드 품질 검사
Test
테스트 러너를 통한 테스트 실행
최소 1개 이상의 테스트가 실행되어야 함
Build
Nest.js 애플리케이션 빌드
빌드 실패 시 파이프라인 전체 실패 처리
📌 주의사항

CI 도구는 자유 선택 (GitHub Actions 기준 권장)
각 단계는 독립적인 job 또는 step으로 명확히 구분되어야 합니다
단순 echo 스크립트는 인정하지 않습니다
 
3️⃣ Docker 기반 컨테이너화
📍 Dockerfile
Nest.js 애플리케이션을 실행할 수 있는 Dockerfile을 작성합니다.

Node.js 기반 이미지 사용
의존성 설치 → 빌드 → 실행 단계가 명확히 분리되어야 함
불필요한 파일이 이미지에 포함되지 않도록 구성
📍 .dockerignore
다음과 같은 항목을 반드시 제외해야 합니다.

node_modules
테스트 결과물
로컬 설정 파일
기타 컨테이너 실행에 불필요한 리소스
 
4️⃣ docker-compose 구성 (DB 포함)
📍 필수 조건
2개의 서비스를 구성해야 합니다
Nest.js 애플리케이션
MySQL 데이터베이스
📍 MySQL 요구사항
MySQL 공식 이미지 사용
환경 변수로 DB 설정 구성
볼륨을 통한 데이터 영속성 유지
📌 주의사항

실제 DB 연동 로직은 구현하지 않아도 됩니다
컨테이너 간 네트워크 연결 구조가 핵심 평가 대상입니다
